# 김치 프리미엄 봇 실행 체크리스트

## ⚠️ 실행 전 필수 확인사항

### 1. **자금 확인**
- [ ] 업비트: 최소 150,000원 이상 (3포지션 × 50,000원)
- [ ] 빗썸: 최소 150,000원 이상 (사용 시)
- [ ] OKX: 최소 $150 이상 (3포지션 × $50)
- [ ] Gate.io: 최소 $150 이상 (3포지션 × $50)

### 2. **API 권한 확인**
- [ ] 업비트: 입출금 및 거래 권한
- [ ] 빗썸: 거래 권한
- [ ] OKX: 현물 및 선물 거래 권한
- [ ] Gate.io: 현물 및 선물 거래 권한

### 3. **봇 설정 확인**
```python
# config.py 확인
TRADING_CONFIG = {
    'max_positions': 3,              # 최대 포지션 수
    'position_size_usd': 50,         # 포지션당 크기
    'split_orders': 5,               # 분할 주문 수
    'order_size_usd': 10,            # 주문당 크기
    'entry_premium_threshold': -0.4,  # 진입 조건
    'exit_premium_threshold': 0.4,    # 청산 조건
}
```

### 4. **위험 관리**
- [ ] 총 투자금의 10% 이하로 설정
- [ ] 네트워크 안정성 확인
- [ ] 긴급 정지 방법 숙지 (Ctrl+C)

### 5. **모니터링**
- [ ] 로그 파일 위치 확인
- [ ] 포지션 상태 주기적 확인
- [ ] 거래소 잔고 확인

## 🚀 실행 명령어

### 기본 실행
```bash
python3 main.py
```

### 백그라운드 실행 (Linux/Mac)
```bash
nohup python3 main.py > bot.log 2>&1 &
```

### 로그 확인
```bash
tail -f bot.log
```

### 프로세스 확인
```bash
ps aux | grep main.py
```

### 봇 중지
```bash
# 프로세스 ID 찾기
ps aux | grep main.py
# 프로세스 종료
kill -9 [PID]
```

## 📊 실시간 모니터링

봇 실행 중 다음 정보를 확인할 수 있습니다:
- 현재 활성 포지션 수
- 각 포지션의 진입 가격 및 프리미엄
- 실시간 프리미엄 변화
- 청산 조건 도달 여부

## ⚠️ 주의사항

1. **실제 돈이 거래됩니다!** 테스트를 충분히 하고 시작하세요.
2. **작은 금액으로 시작**하여 봇이 정상 작동하는지 확인하세요.
3. **네트워크 끊김**에 대비하여 포지션을 주기적으로 확인하세요.
4. **거래소 점검 시간**을 확인하고 그 시간에는 봇을 중지하세요.
5. **세금 신고** 의무가 있을 수 있으니 거래 내역을 보관하세요.

## 🆘 문제 발생 시

1. **봇 즉시 중지**: Ctrl+C
2. **거래소에서 수동으로 포지션 확인**
3. **로그 파일 확인**으로 문제 파악
4. **필요시 수동으로 포지션 청산**
